# LOS - Lambda Operating System

λ計算を理論的基盤とした新しいOS設計パラダイムを探求するプロジェクト。

---

## プロジェクト概要

- **目的**: 純粋関数型言語Haskellで新しいOS設計パラダイムを探求
- **アプローチ**: 既存概念の翻訳ではなく、λ計算から根本的に再設計
- **言語**: Haskell

## 議論の資料化ルール

### 必須事項

**すべての技術的議論・設計議論は、終了後に必ず資料化すること。**

### 資料化の手順

1. 議論が一段落したら、内容を整理してMarkdownファイルを作成
2. 適切なディレクトリに配置（下表参照）
3. ファイル名は連番プレフィックス＋内容を表す名前（例: `00-philosophy.md`）

### ディレクトリ選択ガイド

| 内容 | 配置先 |
|------|--------|
| 設計思想・アーキテクチャ | `docs/design/` |
| 技術調査・既存研究の分析 | `docs/research/` |
| 重要な設計判断とその根拠 | `docs/decisions/` |
| 実装に関する議論 | 該当するソースコード近くのREADME or `docs/design/` |

### ファイル命名規則

```
[連番]-[内容].md

例:
00-philosophy.md        # 設計哲学
01-core-concepts.md     # コア概念の定義
02-type-system.md       # 型システム設計
```

### 資料に含めるべき内容

- 議論の背景・動機
- 検討した選択肢
- 決定事項とその根拠
- 未解決の課題・今後の検討事項
- 参考文献・リンク
- 作成日・ステータス

## 開発ルール

### コーディング

- Haskellのベストプラクティスに従う
- 型シグネチャは必ず明記
- 純粋関数と効果を持つ関数を明確に分離

### コミット

- 意味のある単位でコミット
- コミットメッセージは日本語OK
